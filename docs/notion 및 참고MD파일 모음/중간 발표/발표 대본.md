# 발표 대본

- ppt
    
    안녕하십니까.
    저희는 맞춤형 공공임대 청약 정보 플랫폼인 ‘청약 따잇’ 프로젝트를 진행하고 있는 팀입니다.
    발표를 시작하겠습니다.
    
    이번 발표의 목차입니다.
    
    저희 팀 구성을 소개드리겠습니다.
    PDF 및 AI 파싱과 데이터 수집 (잠깐쉬고)
    프론트엔드 백엔드 아키텍처 (잠깐쉬고) 
    데이터 관리, 백엔드 지원 으로 구성했습니다. 
    
    이어서 서비스 개요와 개발 전략에 대해 설명드리겠습니다.
    저희가 청약 사이트들을 찾아봤을때 기존의 청약 시스템은 공고문이 복잡하고 정보가 분산되어 있어서,사용자가 직접 조건을 비교하고 판단해야 하는 불편함이 있었습니다.
    그래서 저희 서비스는 청년, 신혼부부, 무주택자, 고령자 등 공공임대 수혜 대상자들이 자신에게 맞는 청약 정보를
    쉽고 빠르게 확인할 수 있도록 돕는 서비스를 목표로 하고있습니다.
    특히 "내가 이 공고에 신청 가능한지", "당첨 가능성이 있는지" 같은 부분을 사용자가 판단하는게 아니라 AI 기반 분석 기능을 활용해 사용자 맞춤형 공고 추천과 당첨 가능서 분석 기능을 제공하는 서비스를 기획하게 되었습니다.
    
    다음으로 개발 전략에 대해 설명드리겠습니다.
    저희는 프로젝트를 개발할 때
    바로 전체 시스템에 기능을 적용하는 방식이 아니라,
    먼저 Demo 환경에서 기능을 검증한 뒤 정식 시스템에 통합하는 구조로 개발을 진행하고 있습니다.
    먼저 1단계에서는
    외부 API 연동이나 PDF + AI 파싱 같은 비교적 복잡하고 구현 난도가 높은 기능들을 Demo 환경에서 먼저 개발했습니다.
    그 다음 2단계에서는
    구현한 기능의 정확성과 안정성을 테스트하면서 기능 검증을 진행했습니다.
    마지막으로 3단계에서는
    검증이 완료된 기능들을 정식 백엔드인 subscrition 프로젝트에 플러그인 형태로 통합하고 있습니다.
    
    공통 기술 스택은 다음과 같습니다. (잠깐 쉬고 넘김)
    
    - 발표할때는 안쓰는 내용
        
        다음은 공통 기술 스택입니다.
        백엔드는 Java 17과 Spring Boot 기반으로 개발하고 있으며,
        데이터베이스는 MySQL과 Spring Data JPA를 사용하고 있습니다.
        또한 추후 성능 최적화를 위해 Redis 캐싱도 적용할 예정입니다.
        보안 부분에서는 JWT와 Spring Security를 적용해 사용자 인증 구조를 구현하고 있고,
        Swagger UI를 통해 API 테스트와 문서화를 함께 진행하고 있습니다.
        프론트엔드는 정식 서비스 기준으로 React와 TypeScript를 사용하고 있으며,
        Demo 환경에서는 빠른 기능 검증을 위해 Vanilla JS 기반으로 개발했습니다.
        추가적으로 PDF 데이터 처리를 위해 Apache PDFBox와 AI 파싱 기능을 함께 활용하고 있습니다.
        
    
    다음은 프로젝트 구조입니다. 
    
    프론트엔드, 정식 백엔드, 데모 버전, UI 컴포넌트 라이브러리, PDF 파싱 AI기능, 프로젝트 문서로 이루어져 있습니다.
    
    - 발표할때는 안쓰는 내용
        
        프로젝트는 기능별로 모듈을 분리해서 관리하고 있습니다.
        frontend 폴더에서는 사용자 UI를 담당하고 있고,
        subscrition 폴더는 Spring Boot 기반의 정식 백엔드 서버입니다.
        그리고 demo 폴더에서는 핵심 기능들을 먼저 검증하고 있으며,
        pdf-ai-poc 폴더에서는 PDF 파싱과 AI 기능을 별도로 테스트하고 있습니다.
        또한 Design System 폴더을 통해 UI 컴포넌트를 관리하고 있고,
        docs 폴더에서는 프로젝트 문서를 정리하고 있습니다.
        
    
    다음은 개발 진행 현황입니다.
    현재 요구사항 분석과 Demo 기반 PDF·AI 기능 개발은 완료된 상태입니다.
    시스템 아키텍처 설계와 데이터베이스 설계도 대부분 완료되었으며,
    현재는 정식 백엔드와 프론트엔드 개발을 중심으로 프로젝트를 진행하고 있습니다.
    다음은 프로젝트의 주요 기능입니다.
    첫 번째로, 사용자 조건에 맞는 공고만 제공하는 맞춤형 공고 알림 기능입니다.
    두 번째는 AI 기반 당첨 가능성 분석 기능으로,
    사용자의 조건을 바탕으로 당첨 가능성을 예측합니다.
    마지막으로 외부 API와 PDF 데이터를 자동으로 분석해
    공고 정보를 추출하고 제공하는 기능을 구현하고 있습니다.
    
    다음은 PDF 파싱 파이프라인입니다.
    먼저 공공청약 API를 통해 공고 데이터를 받아오고,
    JSON 형태의 기본 정보를 파싱합니다.
    이후 공고문 PDF를 다운로드해서 AI 기반 텍스트 분석을 진행하고,
    JSON 데이터와 PDF 분석 결과를 통합해 최종 데이터를 구성합니다.
    최종적으로 가공된 정보를 사용자 UI에 제공하는 방식으로 동작합니다.
    특히 JSON 파싱과 AI PDF 분석을 함께 활용해
    기존 단순 API 방식보다 정보 정확도와 완성도를 높인 것이 핵심입니다.
    
    다음은 기술 도전 과제와 해결 방안입니다.
    공고문마다 형식이 달라 PDF 파싱의 복잡성이 큰 문제가 있었는데,
    Apache PDFBox와 AI 기반 분석을 활용해 해결하고 있습니다.
    또한 AI 분석 정확도를 높이기 위해 실제 데이터를 지속적으로 수집하고 있으며,
    대용량 데이터 처리를 위해 Redis 캐싱과 DB 최적화도 함께 진행하고 있습니다.
    외부 API 연동 기능 역시 Demo 환경에서 먼저 검증한 뒤 정식 시스템에 통합하고 있습니다
    
    다음은 향후 계획입니다.
    현재 Q1 단계에서는 Demo 기능 완성과 백엔드 아키텍처 정리, DB 최적화를 진행하고 있습니다.
    이후 Q2에서는 프론트엔드 UI/UX 구현과 API 통합 테스트, AI 프롬프트 엔지니어링을 진행할 예정이며,
    Q3에서는 알파 버전 출시와 사용자 피드백 수집, 성능 최적화를 목표로 하고 있습니다.
    
    마지막으로 기대효과입니다.
    사용자는 자신에게 맞는 공고를 빠르게 확인할 수 있어
    공고 검색 시간을 크게 줄일 수 있습니다.
    또한 AI 기반 추천을 통해 더 정확한 정보를 제공받을 수 있으며,
    당첨 가능성을 사전에 파악해 보다 효율적인 의사결정을 할 수 있을 것으로 기대하고 있습니다.
    여기까지 발표를 마치고 시연 시작하겠습니다.
    
- 시연
    
    먼저 공고 검색 화면입니다. 사용자는 로그인하지 않아도 전체 공고를 볼 수 있습니다. 왼쪽 필터에서 지역, 공급 유형, 주택 유형, 모집 상태, 카테고리, 보증금과 월세 범위를 선택해 원하는 공고만 조회할 수 있습니다.
    
    상세 페이지에서는 공고의 핵심 정보를 한 화면에서 볼 수 있습니다. 신청 기간, 비용 정보, 공급 정보, 문의처,
    원문 링크를 확인할 수 있고, 로그인한 사용자는 관심 있는 공고를 즐겨찾기에 저장할 수 있습니다.
    
    로그인을 하면 개인화 기능을 사용할 수 있습니다. 로그인 후에는 맞춤추천, 즐겨찾기, 마이페이지 메뉴가 표시됩니다.
    
    이 화면은 프로필 설정 입니다.
    프로필 설정은 맞춤 추천을 위해 사용자의 기본 정보, 자격 조건, 선호 조건을 입력받습니다.
    이 정보는 추천 점수 계산과 자격 조건 필터링에 사용됩니다.
    
    핵심 기능인 맞춤 추천 화면입니다. 사용자의 프로필과 공고의 자격 조건, 선호 조건을 비교해서 매칭 점수를
    계산합니다. 단순히 최신 공고를 보여주는 것이 아니라, 사용자의 조건에 맞는 공고를 우선순위로 보여줍니다.
    예를들어 선택된 신분 유형과 일치 할 경우, 희망 주택 유형과 일치할 경우 매칭 점수가 올라갑니다 
    
    사용자는 관심 있는 공고를 즐겨찾기로 저장할 수 있습니다. 저장된 공고는 즐겨찾기 페이지에서 따로 모아볼 수 있고,
    필요하면 다시 상세 페이지로 이동하거나 즐겨찾기를 해제할 수 있습니다.
    
    즐겨찾기 된 공고는 마감 7,3,1일 기준으로 메일이 발송됩니다. 
    
    관리자 계정으로 로그인하면 관리자 메뉴가 추가로 표시됩니다. 백엔드에서는 JWT 안의 권한 정보를 기반으로
    일반 사용자와 관리자를 구분하고, 관리자 API는 ADMIN 권한이 있어야 접근할 수 있습니다.
    
    관리자 대시보드에서는 AI 파싱 검수 상태를 한눈에 확인할 수 있습니다. 검수 대기, 승인, 수정 확정, 거절, 재수집 상태별 공고 수를 볼 수 있고, 검수 목록이나 수동 공고 등록, LH 공고 수집 화면으로 이동할 수 있습니다.
    
    관리자는 LH API에서 공고 데이터를 수집할 수 있습니다. 백엔드는 LH 목록 API를 호출하고, 공고 상세 정보와 첨부
    PDF를 가져옵니다. 이후 PDF 텍스트를 추출하고 Gemini를 통해 자격 조건과 보증금,
    월세, 공급 정보 등을 파싱해서 DB에 저장합니다.
    
    AI가 PDF에서 추출한 결과는 바로 확정하지 않고 검수 대기 상태로 등록됩니다.
    관리자는 목록에서 검수할 공고를 선택할 수 있습니다.
    
    AI 파싱은 편리하지만 항상 정확하지는 않을 수 있습니다 또한 원본 API 데이터가 부실하여 사용할 수 없는 경우도 존재합니다. 그래서 관리자는 바로 승인, 수정, 거절 , 재수집을 할 수 있습니다. 
    
    이 과정을 통해 추천에 사용되는 자격 조건 데이터의 신뢰도를 높일 수 있습니다.
    
    이상으로 발표를 마치겠습니다.
