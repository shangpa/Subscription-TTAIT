# Demo 버전 프론트 기능서 / 화면 설계서 / 디자인 기획서

## 1. 문서 목적

본 문서는 `demo/frontEnd/app` 기준의 데모 프론트를 별도 트랙으로 작업하기 위한 기준 문서다.

이 버전은 **실서비스 구축용이 아니라 시연과 UX 검증용**이다.

---

## 2. 현재 구현 기준

### 기준 경로
- HTML: `demo/frontEnd/app/*.html`
- CSS: `demo/frontEnd/app/assets/css/*`
- JS: `demo/frontEnd/app/assets/js/*`

### 현재 확인된 페이지
- `list.html` : 공고 목록
- `detail.html` : 공고 상세
- `auth.html` : 로그인 / 회원가입
- `mypage.html` : 마이페이지
- `recommend.html` : 추천 공고

### 현재 데이터 방식
- `assets/js/api.js`에서 `USE_MOCK = true`
- `assets/js/mock-data.js` 기반 mock 데이터 사용

---

## 3. demo 버전 목표

### 핵심 목표
1. 서비스 첫인상을 좋게 만든다.
2. 사용자가 3분 안에 전체 플로우를 이해하게 만든다.
3. “공고 탐색 → 상세 확인 → 로그인/가입 → 추천/저장/마이페이지” 흐름을 자연스럽게 보여준다.

### 비목표
- 운영급 예외 처리 완성
- 실제 권한/보안 완성
- 백오피스 운영 기능 완성

---

## 4. 타깃 사용자

### 1차 타깃
- 공공임대/청약 정보를 처음 접하는 사용자
- 서비스 데모를 보는 투자자/협업자/멘토

### 2차 타깃
- UX 검증 대상 일반 사용자
- 향후 실제 서비스 사용 후보군

---

## 5. 핵심 가치 제안

- 복잡한 공공임대 공고를 쉽게 탐색할 수 있다.
- 나에게 맞는 공고를 빠르게 찾을 수 있다.
- 저장과 추천, 알림까지 한 흐름으로 이해할 수 있다.

---

## 6. 정보구조(IA)

```text
목록
 ├─ 필터/정렬
 ├─ 공고 카드
 └─ 상세 진입

상세
 ├─ 공고 핵심 정보
 ├─ 신청 일정
 ├─ 시세 비교
 ├─ 첨부서류
 └─ 신청하기 / 저장하기

인증
 ├─ 로그인
 └─ 회원가입 2단계

마이페이지
 ├─ 프로필
 ├─ 저장한 공고
 └─ 알림 / 알림 설정

추천
 ├─ 추천 요약
 └─ 추천 공고 목록
```

---

## 7. 페이지별 기능서

## 7-1. 공고 목록

### 목적
사용자가 다양한 공고를 빠르게 훑고, 관심 공고를 상세로 진입하게 한다.

### 주요 기능
- 공급유형 필터
- 상태 필터(모집중/마감임박)
- 지역 필터
- 정렬(추천순/마감순/최신순)
- 공고 카드 목록
- 페이지네이션 UI

### 필수 UI 요소
- 상단 메인 헤더
- 공급유형 필터 pill
- 조건 chip row
- 공고 카드 grid
- 결과 수

### 데모에서 중요한 포인트
- 카드 가독성
- 빠른 필터 반응
- “지금 볼 만한 공고”가 바로 눈에 띄는가

### 상태 정의
- 기본 상태
- 필터 적용 상태
- 결과 없음 상태(추가 필요)
- 로딩 상태(추가 필요)

---

## 7-2. 공고 상세

### 목적
공고 한 건의 핵심 판단 정보를 빠르게 전달한다.

### 주요 기능
- 제목/상태/태그 표시
- 공고 기본 정보
- 신청 일정 타임라인
- 시세 비교 카드
- 첨부서류 영역
- 외부 신청 링크
- 저장 버튼

### 필수 UI 요소
- hero 영역
- 상태 배지
- 정보 그리드
- 타임라인
- sticky 요약 카드

### 데모에서 중요한 포인트
- 복잡한 공고문을 “요약된 화면”으로 보여주는 인상
- 신청 일정과 비용이 눈에 잘 들어오는 구조

### 상태 정의
- 모집중
- 마감임박
- 마감
- 분양가형/임대가형 분기

---

## 7-3. 로그인 / 회원가입

### 목적
개인화 추천을 받기 위한 진입 흐름을 짧고 친절하게 제공한다.

### 주요 기능
- 로그인 탭
- 회원가입 탭
- 회원가입 2단계
  - 1단계: 계정 정보
  - 2단계: 맞춤 정보 입력
- 가입 완료 화면

### 데모에서 중요한 포인트
- 가입 절차가 어렵지 않아 보일 것
- 추천 정확도 향상 메시지가 잘 전달될 것

### 상태 정의
- 로그인 기본
- 회원가입 step1
- 회원가입 step2
- 가입 완료
- 입력 오류 상태(추가 필요)

---

## 7-4. 마이페이지

### 목적
개인화 서비스가 실제로 동작하는 느낌을 보여준다.

### 주요 기능
- 프로필 조회/수정형 UI
- 저장한 공고 목록
- 알림 목록
- 알림 설정 토글
- 좌측 사이드 네비게이션

### 데모에서 중요한 포인트
- “저장/알림/개인화”가 하나로 묶여 보일 것
- 실제 서비스처럼 계정이 살아있는 인상을 줄 것

### 상태 정의
- 프로필 탭
- 저장 공고 탭
- 알림 탭
- 읽지 않은 알림 강조

---

## 7-5. 추천 공고

### 목적
서비스의 차별점인 개인화 추천을 가장 직관적으로 보여준다.

### 주요 기능
- 사용자 선호 요약
- 추천 사유 노출
- 추천 점수 표시
- 추천 공고 카드 목록

### 데모에서 중요한 포인트
- 왜 이 공고가 추천되었는지 설명이 보여야 함
- 추천이 단순 목록이 아니라 “개인화 결과”처럼 느껴져야 함

---

## 8. 디자인 기획

## 8-1. 디자인 방향
- 친근함
- 생활형 서비스 느낌
- 복잡한 정책/공고 정보를 부드럽게 풀어주는 인터페이스

## 8-2. 키워드
- 따뜻함
- 신뢰감
- 명확함
- 부담 없는 탐색

## 8-3. 시각 원칙
- 카드 중심 레이아웃
- 짧은 정보 단위로 분절
- 배지/칩으로 상태를 즉시 인식
- 주요 CTA는 항상 명확하게 1~2개만 노출

## 8-4. 핵심 컴포넌트
- 헤더
- 필터 pill / chip
- 공고 카드
- 상태 배지
- 타임라인
- 정보 그리드
- sticky 요약 카드
- 토글 스위치

## 8-5. 카피 톤
- 행정 문구 그대로 노출하지 말고, 사용자 언어로 1차 번역
- 예: “applicationEndDate” 대신 “마감일”, “오늘 꼭 확인” 같은 행동 유도 문구 사용

## 10. 개발 가이드

### 유지 권장 사항
- 현재 정적 파일 구조 유지
- mock 데이터 우선 유지
- 시연 중 끊기지 않도록 네트워크 의존 최소화

### 개선 권장 사항
- `api.js`를 mock adapter / real adapter 형태로 분리
- empty/loading/error 상태 UI 추가
- 페이지별 더미 데이터 일관성 강화
- 발표 시나리오 기준 샘플 사용자 2~3종 추가

---

## 11. 완료 기준(Definition of Done)

- 첫 진입부터 추천 화면까지 흐름이 끊기지 않는다.
- 발표자가 클릭할 주요 동선이 명확하다.
- 모든 핵심 페이지가 공통 톤으로 연결된다.
- mock 데이터만으로도 제품 가치가 충분히 전달된다.
