# 프론트 작업 분리 가이드

## 1. 목적

demo 버전과 subscription 메인 버전이 서로 발목 잡지 않고, **별도 일정 / 별도 품질 기준 / 별도 산출물**로 프론트 작업 가능하도록 기준을 정의한다.

---

## 2. 제품 포지션 분리

### A. demo 버전
- 목적: 투자자/팀 내부/사용자 테스트용 시연
- 우선순위: 속도, 화면 완성도, 스토리텔링, UX 검증
- 데이터 기준: mock 우선, 일부 API 연결 가능
- 기술 기준: 현재 구조 유지 가능 (정적 HTML/CSS/Vanilla JS)
- 성공 기준: “서비스가 어떻게 동작하는지 한 번에 이해된다”

### B. 메인 버전
- 목적: 실제 운영 가능한 서비스 구축
- 우선순위: 안정성, API 정합성, 인증/권한, 유지보수성
- 데이터 기준: Spring Boot API 실연동
- 기술 기준: React + TypeScript 기반 신규 프론트 권장
- 성공 기준: “실제 가입/로그인/조회/수정/관리자 검수까지 운영 가능하다”

---

## 3. 병렬 작업 원칙

### 원칙 1. 저장소/폴더 레벨 분리
- demo 작업 기준 폴더: `demo/frontEnd/app`
- main 작업 기준 폴더: 신규 `frontend/` 또는 `subscrition/frontend/` 권장
- 두 버전은 CSS/JS 파일을 공유하지 않는다.

### 원칙 2. 디자인 시스템은 “규칙만 공유”한다
- 컬러 톤
- 타이포 위계
- 카드/배지/버튼 스타일 방향성
- 용어 체계

즉, **시각 언어는 공유 가능하지만 코드 자산은 분리**한다.

### 원칙 3. 요구사항 수준이 다르다
- demo: happy path 중심
- main: 예외/빈 상태/권한/로딩/실패/재시도 포함

### 원칙 4. 화면 이름은 최대한 맞춘다
- 목록
- 상세
- 로그인/회원가입
- 마이페이지
- 추천
- 관리자 검수

화면 이름을 맞춰두면 demo에서 검증한 UX를 main으로 옮기기 쉽다.

---

## 4. 역할 분담 추천

### 트랙 A. demo 프론트 담당
- 랜딩/목록/상세/추천의 시연 흐름 고도화
- mock 데이터 품질 개선
- 발표용 시나리오 구성
- 시각 polish
- 사용성 테스트 반영

### 트랙 B. main 프론트 담당
- React 앱 초기 세팅
- API 클라이언트/인증 구조 설계
- 실제 화면 구현
- 관리자 화면 구현
- 배포/환경변수/에러 처리 구축

### 트랙 C. 공통 기획/디자인 담당
- 메뉴 구조 정리
- 용어 사전 관리
- 공통 컴포넌트 규칙 정리
- demo → main 이관 우선순위 결정

---

## 5. 공통 산출물 포맷

각 버전은 아래 4개를 기본 산출물로 가진다.

1. 기능서
   - 어떤 기능을 왜 제공하는지
2. 화면 설계서
   - 페이지 구조, 섹션, 상태, 전환 흐름
3. 디자인 기획서
   - 톤앤매너, 컴포넌트, UX 원칙
4. 개발 범위표
   - 이번 스프린트 포함 / 제외 / 보류

---

## 6. 인수인계 규칙

### demo → main 으로 넘길 수 있는 것
- 검증된 사용자 흐름
- 페이지 IA
- 카드/필터/배지/타임라인 패턴
- 카피 문구 방향
- 우선순위가 검증된 기능 목록

### 그대로 넘기면 안 되는 것
- mock 응답 구조
- 정적 파일 구조
- localStorage 임시 처리
- 임시 하드코딩 데이터

---

## 7. 작업 순서 추천

### 1차
- demo 문서 확정
- main 문서 확정
- 공통 용어/메뉴/디자인 토큰 초안 정리

### 2차
- demo는 발표용 플로우 완성
- main은 React 구조/라우팅/API 레이어 구축

### 3차
- main에 실제 API 연동
- 관리자 검수 기능 연결
- 예외/권한 처리 강화

### 4차
- demo에서 반응 좋았던 UX를 main에 선별 반영

---

## 8. 최종 의사결정 기준

의견이 충돌하면 아래 순서로 결정한다.

1. 메인 운영 안정성
2. 사용자 핵심 과업 성공률
3. demo 시연 전달력
4. 구현 난이도/일정

즉, demo는 자유롭게 실험하되, main은 운영 기준을 우선한다.
