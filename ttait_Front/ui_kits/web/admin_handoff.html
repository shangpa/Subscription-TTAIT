ADMIN DTO FIELD REDESIGN

운영 판단 필드는 바로 보이고,
원문·디버그 값은 접어서 확인합니다.

핸드오프 문서에서 보류했던 P0/P1/P2 필드를 화면 밀도와 혼동 가능성 기준으로 재배치했습니다. 검수자는 목록에서 우선순위를 빠르게 판단하고, 상세에서는 원문 근거를 필요할 때 펼쳐 볼 수 있습니다.

이번 재설계 원칙
  • P0는 운영 판단용으로 기본 노출
  • P1는 요약 영역에 보조 정보로 노출
  • P2는 단위 검수/디버깅 접힘 영역으로 분리
  • null/false는 “미확인/아니오”로 명확히 구분
DESIGN DECISIONS

핸드오프 질문에 대한 프론트 설계 결정

임의 배치가 아니라 운영자가 바로 써야 하는 정보와 검수 근거 정보를 계층화하는 방향입니다.

재설계 반영
P0 processedToday

대시보드 핵심 지표 카드로 승격합니다. 전체 처리량보다 “오늘 처리”를 먼저 보이게 해 당일 운영 속도를 판단합니다.

P0 noticeStatus + reviewStatus

목록에서 공고 상태는 “공고 상태”, 검수 상태는 “검수 상태”로 분리된 칩 컬럼에 배치해 의미 혼동을 줄입니다.

P0 applicationStartDate

마감일 단독 표시를 기간 표시로 변경합니다. D-day는 마감 기준 강조 배지로 유지합니다.

P0 scheduleDetailsJson

JSON 원문을 기본 노출하지 않고 “구조화 일정 카드 + JSON 원문 접힘” 조합으로 제공합니다.

P0 importantNotesRaw

중요 안내는 상세 검수의 별도 강조 카드로 노출하되 긴 원문은 접힘 영역으로 둡니다.

P2 unitId / exclusiveAreaValue / salePriceRaw

목록형 단위 테이블에는 보조 텍스트로 최소 노출하고, 내부 식별자와 원문 가격은 “운영 디버그” 접힘 영역에 둡니다.
