## 상세 테스트 내용

### 1. 권한 검증

#### 미인증 접근
```
GET /api/admin/review/stats
→ HTTP 403
```

#### USER 권한으로 admin API 접근
```
GET /api/admin/review/stats
Authorization: Bearer {USER_TOKEN}
→ HTTP 403

DELETE /api/admin/announcements/4
Authorization: Bearer {USER_TOKEN}
→ HTTP 403
```

SecurityConfig의 `.requestMatchers("/api/admin/**").hasRole("ADMIN")` 정상 작동.

### 5. 검수 액션 — CORRECT *(버그 수정 검증)*

```http
POST /api/admin/review/4
Content-Type: application/json

{
  "action": "CORRECT",
  "note": "나이 조건 수정 테스트",
  "corrections": {
    "ageMin": 19,
    "ageMax": 35,
    "homelessRequired": true
  }
}
```

→ **HTTP 200 OK** ✅

> **버그 수정 전**: CORRECT 액션이 항상 HTTP 403을 반환했음.
>
> **원인**: `handleCorrect()` 내에서 예외 발생 시 GlobalExceptionHandler가 처리하지 못하고
> Spring Boot가 `/error`로 포워딩. STATELESS 세션 정책에서 에러 디스패치 시 인증 컨텍스트가
> 끊겨 Spring Security가 403을 반환하는 구조.
>
> **수정 내용**:
> 1. `SecurityConfig.java`: `.requestMatchers("/error").permitAll()` 추가
> 2. `GlobalExceptionHandler.java`: `@ExceptionHandler(Exception.class)` catch-all 핸들러 추가 → 500 + 에러 메시지 반환

### 8. 검수 액션 — REIMPORT

```http
POST /api/admin/review/6
{ "action": "REIMPORT" }
```

→ **HTTP 404** `{"message": "LH_ITEM_JSON not found for announcement: 6"}`

> ⚠️ **정상 에러 응답** — 해당 공고의 `announcement_parse_raw` 테이블에 `LH_ITEM_JSON` 타입
> 원본 데이터가 없어서 LH API 재수집 불가. 실제 LH API 수집으로 생성된 공고에만 REIMPORT 가능.
>
> **버그 수정 전**: 이 에러도 예외가 처리되지 않아 403을 반환했음.
> 수정 후 적절한 404 + 에러 메시지 반환으로 개선됨.

### 11. 테스트 완료 후 최종 통계 (버그 수정 전)

```json
{
  "pending": 2,
  "approved": 2,
  "corrected": 0,
  "rejected": 1,
  "reImport": 0,
  "totalAnnouncements": 4,
  "processedToday": 3
}
```

- PENDING 2: ID 7 (기존), ID 8 (수동 등록)
- APPROVED 2: ID 4 (CORRECT→APPROVE 순서로 처리), ID 6 (REIMPORT 실패로 APPROVED 유지)
- REJECTED 1: ID 5 (소프트 삭제된 공고 포함 — 버그)
- totalAnnouncements 4: ID 4, 6, 7, 8 (ID 5는 deleted=true로 제외)

## 발견된 버그 및 이슈

### 버그 1 — 소프트 삭제 공고가 상태별 카운트에 포함됨 ✅ 수정 완료

- **증상**: ID 5 공고가 소프트 삭제(`deleted=true`)된 후에도 stats의 `rejected` 카운트에 계속 포함됨
- **원인**: `AnnouncementEligibilityRepository.countByReviewStatus()`, `countByReviewedAtGreaterThanEqualAndReviewStatusNot()` 파생 메서드가 `announcement.deleted` 조건 미필터
- **수정**: 두 메서드를 `@Query`로 교체, `AND e.announcement.deleted = false` 조건 추가

**수정 후 재테스트 결과**:
```json
{
  "pending": 2,
  "approved": 2,
  "corrected": 0,
  "rejected": 0,
  "reImport": 0,
  "totalAnnouncements": 4,
  "processedToday": 2
}
```
`rejected: 1 → 0` — 삭제된 ID 5가 카운트에서 제외됨. `totalAnnouncements: 4`와 일관성 유지됨.
