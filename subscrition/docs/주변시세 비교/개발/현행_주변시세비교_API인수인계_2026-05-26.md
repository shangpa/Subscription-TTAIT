# [현행] 주변시세 비교 API 인수인계 2026-05-26

<!-- markdownlint-disable MD013 -->

## 1. 문서 목적

이 문서는 주변시세 비교 backend 작업을 프론트 담당자와 팀원 AI가 이어받기 쉽도록 정리한 인수인계 문서다.
현재 범위는 RTMS raw 수집, `MarketPriceSnapshot` 집계, 공고 단위별 주변시세 비교 API, 법정동 seed import, RTMS/snapshot batch API까지다.
프론트 화면 구현과 지도 marker 렌더링은 이 문서의 완료 범위가 아니다. scheduler형 완전 자동화는 후속 정책으로 남기되, 관리자 batch API는 추가됐다.

## 3. 전체 데이터 흐름

```text
LH 공고 수집
→ AnnouncementUnit 저장
→ 법정동 seed import 또는 기존 mapping 준비
→ unit 주소 정규화로 legalDongCode/lawdCd 저장
→ RTMS raw 수집 또는 RTMS/snapshot batch 실행
→ market_transaction_raw 저장
→ market_price_snapshot 집계
→ 프론트가 공고 unit 기준 비교 API 호출
→ unit 임대료/분양가와 snapshot 평균값 비교 응답
```

## 5. Snapshot 집계 API

### 5.1 Request

```http
POST /api/admin/market/snapshots/aggregate
Content-Type: application/json
```

```json
{
  "sourceType": "APT_RENT",
  "lawdCd": "41570",
  "dealYmFrom": "202401",
  "dealYmTo": "202406",
  "areaMin": 50.0,
  "areaMax": 70.0,
  "minimumSampleCount": 3
}
```

| 필드 | 설명 | 필수 |
| - | - | - |
| `sourceType` | RTMS/market source type. 예: `APT_RENT`, `APT_TRADE` | Y |
| `lawdCd` | 법정동 코드 앞 5자리 | Y |
| `dealYmFrom` | 집계 시작 계약년월 `YYYYMM` | Y |
| `dealYmTo` | 집계 종료 계약년월 `YYYYMM` | Y |
| `areaMin` | 집계 면적 하한 | Y |
| `areaMax` | 집계 면적 상한 | Y |
| `minimumSampleCount` | `OK` 판단 최소 sample 수. 생략 시 3 | N |

### 5.2 Response

```json
{
  "snapshotId": 7,
  "sourceType": "APT_RENT",
  "lawdCd": "41570",
  "dealYmFrom": "202401",
  "dealYmTo": "202406",
  "areaMin": 50.00,
  "areaMax": 70.00,
  "sampleCount": 3,
  "avgDepositAmount": 70000,
  "medianDepositAmount": 70000,
  "avgMonthlyRentAmount": 30,
  "medianMonthlyRentAmount": 30,
  "avgTradeAmount": null,
  "medianTradeAmount": null,
  "status": "OK",
  "snapshotKey": "...sha256...",
  "aggregatedAt": "2026-05-26T12:00:00"
}
```

### 5.3 집계 정책

- `market_transaction_raw`에서 `sourceType`, `lawdCd`, `dealYmFrom~dealYmTo`, `areaMin~areaMax`에 맞는 row를 조회한다.
- `sampleCount >= minimumSampleCount`이면 `status=OK`다.
- sample 수가 부족하거나 0건이면 snapshot은 저장하되 `status=INSUFFICIENT_DATA`로 응답한다.
- 같은 조건의 snapshot은 같은 `snapshotKey`를 사용하며, 재집계 시 기존 snapshot을 삭제 후 새로 저장한다.
- 평균은 반올림 정수, 중앙값은 짝수 개일 때 가운데 두 값 평균의 반올림 정수다.

## 6. 프론트용 주변시세 비교 API

### 6.1 Request

```http
GET /api/announcements/10/units/20/market-comparison?sourceType=APT_RENT&dealYmFrom=202401&dealYmTo=202406
```

| 파라미터 | 설명 | 필수 |
| - | - | - |
| `announcementId` | 공고 ID path variable | Y |
| `unitId` | `AnnouncementUnit` ID path variable | Y |
| `sourceType` | 비교할 market source type | Y |
| `dealYmFrom` | 비교 snapshot 기간 시작 `YYYYMM` | Y |
| `dealYmTo` | 비교 snapshot 기간 종료 `YYYYMM` | Y |

현재 API는 unit의 `lawdCd`와 `exclusiveAreaValue`를 기준으로 이미 생성된 snapshot을 찾는다.
따라서 프론트 호출 전에 해당 unit의 주소 정규화와 snapshot 집계가 먼저 되어 있어야 한다.

### 6.2 Response 예시: 비교 가능

```json
{
  "announcementId": 10,
  "unitId": 20,
  "sourceType": "APT_RENT",
  "lawdCd": "41570",
  "dealYmFrom": "202401",
  "dealYmTo": "202406",
  "exclusiveArea": 59.84,
  "status": "COMPARABLE",
  "message": null,
  "unitPrice": {
    "depositAmount": 75000,
    "monthlyRentAmount": 35,
    "salePriceMin": null,
    "salePriceMax": null
  },
  "snapshot": {
    "snapshotId": 30,
    "sampleCount": 5,
    "snapshotStatus": "OK",
    "areaMin": 50.00,
    "areaMax": 70.00,
    "avgDepositAmount": 70000,
    "medianDepositAmount": 71000,
    "avgMonthlyRentAmount": 30,
    "medianMonthlyRentAmount": 31,
    "avgTradeAmount": null,
    "medianTradeAmount": null,
    "aggregatedAt": "2026-05-26T12:00:00"
  },
  "depositComparison": {
    "unitAmount": 75000,
    "marketAmount": 70000,
    "differenceAmount": 5000,
    "differenceRatePercent": 7.14
  },
  "monthlyRentComparison": {
    "unitAmount": 35,
    "marketAmount": 30,
    "differenceAmount": 5,
    "differenceRatePercent": 16.67
  },
  "tradeComparison": null
}
```

### 6.3 Response 해석

| 필드 | 의미 |
| - | - |
| `unitPrice` | LH 공고 unit에 저장된 가격 정보 |
| `snapshot` | 비교에 사용한 `market_price_snapshot` 요약 |
| `depositComparison` | unit 보증금과 snapshot 평균 보증금 비교 |
| `monthlyRentComparison` | unit 월세와 snapshot 평균 월세 비교 |
| `tradeComparison` | 분양/매매 금액 비교. 임대 source에서는 보통 null |
| `differenceAmount` | `unitAmount - marketAmount` |
| `differenceRatePercent` | `(unitAmount - marketAmount) / marketAmount * 100` |

`differenceAmount`가 양수면 공고 unit 가격이 snapshot 평균보다 높은 것이다.
음수면 공고 unit 가격이 snapshot 평균보다 낮은 것이다.

## 7. Status와 실패/빈 데이터 케이스

### 7.1 Snapshot status

| status | 의미 | 프론트 처리 제안 |
| - | - | - |
| `OK` | 최소 sample 수 이상으로 비교 신뢰 가능 | 비교 수치 표시 |
| `INSUFFICIENT_DATA` | raw 거래가 없거나 sample 수 부족 | 수치는 참고용 또는 안내 문구 표시 |

### 7.2 비교 API status

| status | 의미 | 프론트 처리 제안 |
| - | - | - |
| `COMPARABLE` | snapshot이 있고 sample도 충분함 | 비교 수치 표시 |
| `INSUFFICIENT_DATA` | snapshot은 있으나 sample이 부족함 | “참고 데이터 부족” 표시 |
| `SNAPSHOT_NOT_FOUND` | 조건에 맞는 snapshot 없음 | 관리자 수집/집계 필요 안내 |
| `UNIT_LAWD_CD_MISSING` | unit에 `lawdCd` 없음 | 주소 정규화 필요 |
| `UNIT_AREA_MISSING` | unit에 전용면적 값 없음 | 비교 불가 표시 |

### 7.3 HTTP error

| 상황 | 응답 |
| - | - |
| 존재하지 않는 `announcementId/unitId` 조합 | `404`, `announcement unit not found` |
| `sourceType` 누락 | `400`, `sourceType is required` |
| `dealYmFrom/dealYmTo` 형식 오류 | `400`, `dealYmFrom must be YYYYMM` 등 |
| `dealYmFrom > dealYmTo` | `400`, `dealYmFrom must be before or equal to dealYmTo` |

```http

```

```json

    {
      "regionLevel1": "경기도",
      "regionLevel2": "김포시",
      "legalDongName": "마산동",
```

```json

  "insertedCount": 1,
  "updatedCount": 0

```

`legalDongCode`는 10자리 숫자여야 하며, `lawdCd`는 앞 5자리에서 자동 계산된다.

### 12.2 공고 unit 주소 정규화 재처리

```http
POST /api/admin/market/address/announcements/1/normalize-units
Content-Type: application/json
```

```json
{
  "retryNoLawdCode": true
}
```

응답:

```json
{
  "announcementId": 1,
  "processedCount": 3,
  "successCount": 2,
  "noAddressCount": 0,
  "noLawdCodeCount": 1
}
```

- `retryNoLawdCode=false` 또는 생략 시 `NOT_REQUESTED` unit만 처리한다.
- `retryNoLawdCode=true`면 기존 `NO_LAWD_CODE` unit도 다시 처리한다.
- LH import/reimport 후에는 `NOT_REQUESTED` unit 주소 정규화가 자동으로 실행된다.


## 13. RTMS 전체 page 수집 API

기존 단일 page API는 유지하고, `totalCount` 기반 전체 page 수집 API를 추가했다.

```http
POST /api/admin/market/rtms/collect-all
Content-Type: application/json
```

```json
{
  "sourceType": "APT_RENT",
  "lawdCd": "28200",
  "dealYm": "202405",
  "numOfRows": 100,
  "maxPages": 100
}
```

응답:

```json
{
  "sourceType": "APT_RENT",
  "lawdCd": "28200",
  "dealYm": "202405",
  "status": "SUCCESS",
  "fetchedCount": 150,
  "savedCount": 140,
  "duplicateCount": 10,
  "failedCount": 0,
  "collectedPageCount": 2,
  "totalCount": 150,
  "message": null
}
```

정책:

- `totalCount`가 있으면 `pageNo * numOfRows >= totalCount`일 때 중단한다.
- `totalCount`가 없더라도 현재 page의 item 수가 `numOfRows`보다 작으면 마지막 page로 보고 중단한다.
- 첫 page가 `NO_RESULT`면 저장 없이 `NO_RESULT`를 반환한다.
- 중간 page에서 `NO_RESULT`가 나오면 이전 page까지의 수집 결과를 `SUCCESS`로 반환한다.
- RTMS 응답 `resultCode=03` 또는 `INFO-200`은 실패가 아니라 `NO_RESULT`로 처리한다.

## 14. 후속으로 남길 작업

- 최신 공공 법정동 코드 전체 파일을 운영/개발 환경에서 seed import한다.
- 괄호 법정동이 없는 도로명주소는 Juso API 또는 geocoding 기반 법정동 보강을 검토한다.
- scheduler형 자동 운영이 필요하면 batch service를 cron 정책에 연결한다.
- 비교 기준 고도화: 면적 허용 오차, 최근 N개월 default, 주택 유형 자동 매핑.
- 프론트 화면 구현: 카드/리스트/지도 marker에서 `status`별 표시 정책 적용.
