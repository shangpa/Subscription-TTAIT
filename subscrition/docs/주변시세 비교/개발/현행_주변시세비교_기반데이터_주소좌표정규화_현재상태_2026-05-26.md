# [현행] 주변시세 비교 기반 데이터 주소/좌표 정규화 현재 상태 2026-05-26

<!-- markdownlint-disable MD013 -->

## 1. 문서 목적

이 문서는 이전 `좌표/Geocoding 1차 MVP` 문서를 현재 main 기준으로 대체한다.
현재는 좌표 저장뿐 아니라 법정동 seed import, 주소 정규화, `lawdCd` 저장까지 주변시세 비교 선행 데이터 흐름에 포함되어 있다.

## 2. 현재 기준 데이터 보강 흐름

```text
LH import/reimport
→ AnnouncementUnit 저장
→ AddressNormalizationService로 legalDongCode/lawdCd 보강
→ AnnouncementUnitGeocodingEnrichmentService로 latitude/longitude 보강
→ 주변시세 비교 API의 선행 데이터 준비
```

`NoticeImportOrchestrator`에서 주소 정규화 후 geocoding 보강이 실행된다.
외부 API 실패는 import 전체 실패로 전파하지 않는 best-effort 흐름을 유지한다.

## 3. AnnouncementUnit에 쌓이는 핵심 필드

| 필드 | 역할 |
| - | - |
| `fullAddress` | LH/PDF에서 추출한 원주소 |
| `regionLevel1`, `regionLevel2` | 시도/시군구 수준 주소 정보 |
| `exclusiveAreaValue` | snapshot 매칭 기준 면적 |
| `depositAmount`, `monthlyRentAmount` | 임대 비교 기준 금액 |
| `salePriceMin`, `salePriceMax` | 분양/매매성 비교 시 참고 금액 |
| `legalDongCode` | 10자리 법정동 코드 |
| `lawdCd` | RTMS 조회용 5자리 지역 코드 |
| `addressStatus` | 주소 정규화 상태 |
| `latitude`, `longitude` | 지도/위치 표시용 좌표 |
| `geocodeStatus` | Naver geocoding 상태 |

## 4. 법정동 매핑 기준

`lawd_code_mapping`은 다음 조합을 기준으로 조회한다.

```text
regionLevel2 + legalDongName + active=true
```

예시:

| regionLevel1 | regionLevel2 | legalDongName | legalDongCode | lawdCd |
| - | - | - | - | - |
| 인천광역시 | 부평구 | 부평동 | 2823710100 | 28237 |
| 인천광역시 | 남동구 | 논현동 | 2820011000 | 28200 |

공공 seed import는 다음 endpoint로 처리한다.

```http
POST /api/admin/market/address/lawd-code-mappings/import
```

## 5. 주소 parsing 현재 동작

`AddressNormalizationService`는 현재 다음 케이스를 처리한다.

| 주소 예시 | regionLevel2 | legalDongName | 비고 |
| - | - | - | - |
| `인천광역시 부평구 부평동 297` | 부평구 | 부평동 | 일반 지번 주소 |
| `경기도 수원시 영통구 이의동 1` | 영통구 | 이의동 | 시 + 구가 같이 있는 주소는 더 구체적인 구 선택 |
| `인천광역시 남동구 은봉로 297(논현동)` | 남동구 | 논현동 | 괄호 안 법정동 우선 |

현재 한계:

- `인천광역시 남동구 은봉로 297`처럼 괄호 법정동이 없는 도로명주소는 법정동을 확정하지 못할 수 있다.
- 이 경우 `NO_LAWD_CODE`가 될 수 있고, Juso API 또는 geocoding 기반 보강이 필요하다.

## 6. 상태 의미

| addressStatus | 의미 |
| - | - |
| `NOT_REQUESTED` | 아직 주소 정규화 전 |
| `SUCCESS` | `legalDongCode`, `lawdCd` 저장 완료 |
| `NO_ADDRESS` | 주소 자체가 없음 |
| `NO_LAWD_CODE` | 주소는 있으나 매핑을 찾지 못함 |

| geocodeStatus | 의미 |
| - | - |
| `NOT_REQUESTED` | geocoding 전 |
| `SUCCESS` | 좌표 저장 완료 |
| `NO_RESULT` | Naver 결과 없음 |
| `FAILED` | Naver/API/runtime 실패 |
| `SKIPPED_NO_ADDRESS` | 주소 없음으로 skip |
