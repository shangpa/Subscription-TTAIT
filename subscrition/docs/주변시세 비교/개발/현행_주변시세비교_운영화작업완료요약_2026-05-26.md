## 3. 추가된 endpoint

### 3.1 공공 법정동 코드 seed import

```http
POST /api/admin/market/address/lawd-code-mappings/import
Authorization: Bearer {ADMIN_TOKEN}
Content-Type: application/json
```

```json
{
  "delimiter": "tab",
  "activeOnly": true,
  "content": "법정동코드\t법정동명\t폐지여부\n2823710100\t인천광역시 부평구 부평동\t존재"
}
```

응답 예시:

```json
{
  "dataLineCount": 1,
  "parsedCount": 1,
  "skippedCount": 0,
  "insertedCount": 1,
  "updatedCount": 0
}
```

주의:

- `content`에는 공공 법정동 코드 CSV/TSV 파일 내용을 그대로 넣는다.
- header는 `법정동코드`, `법정동명`, `폐지여부`를 우선 인식한다.
- `activeOnly=true`이면 폐지 row는 skip한다.
- 시/구가 함께 있는 주소는 더 구체적인 `구`를 `regionLevel2`로 저장한다.

### 3.2 RTMS 수집 + snapshot 집계 batch

```http
POST /api/admin/market/batches/rtms-snapshot
Authorization: Bearer {ADMIN_TOKEN}
Content-Type: application/json
```

```json
{
  "sourceType": "APT_RENT",
  "lawdCd": "28237",
  "dealYm": "202405",
  "numOfRows": 100,
  "maxPages": 1,
  "dealYmFrom": "202405",
  "dealYmTo": "202405",
  "areaMin": 20.0,
  "areaMax": 30.0,
  "minimumSampleCount": 1
}
```

batch는 내부적으로 다음 순서로 실행된다.

```text
RTMS collect-all
→ market_transaction_raw 저장/중복 skip
→ MarketPriceSnapshot aggregate
→ collection + snapshot summary 응답
```

## 5. 남은 리스크

| 리스크 | 설명 | 후속 방향 |
| - | - | - |
| 도로명주소에 법정동 괄호가 없는 unit | 예: `인천광역시 남동구 은봉로 297`은 현재 road name만으로 법정동을 확정하지 못한다. | 주소 API/Juso API 또는 geocoding 결과 기반 법정동 보강 검토 |
| full seed 실제 운영 적재 | endpoint는 준비됐지만 repository에 공공 전체 원본 파일을 커밋하지는 않았다. | 운영/개발 환경에서 최신 공공 법정동 코드 파일을 내려받아 import 실행 |
| scheduler 자동화 | 이번 작업은 batch API 연결까지다. | cron scheduler가 필요하면 batch service를 scheduler에서 호출하도록 별도 정책화 |
| sample 수 | 이번 E2E는 `minimumSampleCount=1`로 성공 흐름을 검증했다. | 운영 기준에서는 최소 sample 수 정책 재검토 |

## 6. 팀원에게 전달할 현재 기준

팀원에게 넘겨도 되는 backend 기준은 다음과 같다.

- 최신 branch에서 unit/integration test가 통과한다.
- 법정동 seed import API로 매핑을 대량 적재할 수 있다.
- LH import/reimport 후 `normalize-units`로 `AnnouncementUnit.lawdCd`를 재처리할 수 있다.
- RTMS 수집과 snapshot 집계는 batch API 하나로 실행할 수 있다.
- public 비교 API는 snapshot이 있으면 `COMPARABLE`을 반환한다.
- Swagger UI에서 주변시세 관련 요청 예시를 확인할 수 있다.
