# [현행] 주변시세 비교 E2E 수동 테스트 가이드 2026-05-26

<!-- markdownlint-disable MD013 -->

## 1. 테스트 목적

이 문서는 현재 main 기준 주변시세 비교 기능을 local에서 끝까지 확인하는 수동 테스트 가이드다.
이전 좌표/Geocoding 전용 Apidog 가이드는 보관 문서로 내리고, 이 문서를 현재 E2E 기준으로 사용한다.

검증 목표:

1. 관리자 인증으로 법정동 seed import가 되는지 확인한다.
2. LH unit 주소 재정규화로 `lawdCd`가 채워지는지 확인한다.
3. RTMS raw 수집과 snapshot 집계가 batch로 이어지는지 확인한다.
4. public 비교 API가 `COMPARABLE` 또는 명확한 실패 status를 반환하는지 확인한다.

## 2. 사전 준비

```bash
cd ~/dev/Subscription-TTAIT/subscrition

docker compose -f docker-compose.local.yml up -d
SPRING_CONFIG_IMPORT="optional:file:/home/user/dev/Subscription-TTAIT/subscrition/.env[.properties]" bash ./gradlew bootRun
```

필수 환경:

| 설정 | 목적 |
| - | - |
| `DB_URL`, `DB_USERNAME`, `DB_PASSWORD` | MySQL 연결 |
| `RTMS_SERVICE_KEY` | 국토교통부 RTMS API 호출 |
| `NAVER_MAPS_CLIENT_ID`, `NAVER_MAPS_CLIENT_SECRET` | geocoding 확인 시 필요 |
| ADMIN JWT | `/api/admin/**` 호출 |

Swagger:

```text
http://localhost:8080/swagger-ui/index.html
```

## 3. 관리자 token 준비

기존 ADMIN 계정이 있으면 login 후 token을 사용한다.
없으면 임시 사용자를 가입시키고 DB에서 role을 `ADMIN`으로 바꾼 뒤 다시 login한다.

```sql
update users
set role = 'ADMIN'
where login_id = '{loginId}';
```

## 4. 법정동 seed import

```http
POST /api/admin/market/address/lawd-code-mappings/import
Authorization: Bearer {ADMIN_TOKEN}
Content-Type: application/json
```

```json
{
  "delimiter": "tab",
  "activeOnly": true,
  "content": "법정동코드\t법정동명\t폐지여부\n2820011000\t인천광역시 남동구 논현동\t존재\n2823710100\t인천광역시 부평구 부평동\t존재"
}
```

성공 기준:

```text
parsedCount > 0
skippedCount는 폐지/상위 행정구역 row 수에 따라 달라질 수 있음
insertedCount + updatedCount > 0
```

## 5. LH unit 주소 재정규화

```http
POST /api/admin/market/address/announcements/{announcementId}/normalize-units
Authorization: Bearer {ADMIN_TOKEN}
Content-Type: application/json
```

```json
{
  "retryNoLawdCode": true
}
```

DB 확인:

```sql
select id, announcement_id, full_address, legal_dong_code, lawd_cd, address_status,
       exclusive_area_value, deposit_amount, monthly_rent_amount
from announcement_unit
where announcement_id = {announcementId}
order by id;
```

성공 기준:

```text
비교 대상 unit의 address_status = SUCCESS
비교 대상 unit의 lawd_cd is not null
비교 대상 unit의 exclusive_area_value is not null
```

## 6. RTMS/snapshot batch

```http
POST /api/admin/market/batches/rtms-snapshot
Authorization: Bearer {ADMIN_TOKEN}
Content-Type: application/json
```

```json
{
  "sourceType": "APT_RENT",
  "lawdCd": "28237",
  "dealYm": "202405",
  "numOfRows": 100,
  "maxPages": 1,
  "dealYmFrom": "202405",
  "dealYmTo": "202405",
  "areaMin": 20.0,
  "areaMax": 30.0,
  "minimumSampleCount": 1
}
```

성공 기준:

```text
status = SUCCESS
collection.status = SUCCESS 또는 NO_RESULT
snapshotAggregated = true
snapshot.status = OK 또는 INSUFFICIENT_DATA
```

주의:

- `minimumSampleCount=1`은 성공 흐름 검증용이다.
- 운영 기준은 더 높은 sample 기준을 별도 정책으로 정한다.

## 7. public 비교 API

```http
GET /api/announcements/{announcementId}/units/{unitId}/market-comparison?sourceType=APT_RENT&dealYmFrom=202405&dealYmTo=202405
```

성공 예시 핵심:

```json
{
  "announcementId": 1,
  "unitId": 3,
  "sourceType": "APT_RENT",
  "lawdCd": "28237",
  "dealYmFrom": "202405",
  "dealYmTo": "202405",
  "exclusiveArea": 25.77,
  "status": "COMPARABLE"
}
```

## 8. status별 원인 확인

| Status | 확인할 것 |
| - | - |
| `COMPARABLE` | 정상. 비교 수치 표시 가능 |
| `INSUFFICIENT_DATA` | snapshot은 있으나 sample 수가 `minimumSampleCount` 미만 |
| `SNAPSHOT_NOT_FOUND` | 해당 `sourceType/lawdCd/dealYm/area` 조건 snapshot 없음 |
| `UNIT_LAWD_CD_MISSING` | `announcement_unit.lawd_cd` 누락. seed/normalize 확인 |
| `UNIT_AREA_MISSING` | `announcement_unit.exclusive_area_value` 누락. LH/PDF unit 추출 확인 |
