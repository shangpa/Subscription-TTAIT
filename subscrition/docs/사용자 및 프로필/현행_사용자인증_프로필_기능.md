# [현행] 사용자 인증 및 프로필 기능

> 구현 완료 기준일: 2026-05-02  
> 관련 패키지: `com.ttait.subscription.auth`, `com.ttait.subscription.user`

---

## 1. 개요

| 기능 | 상태 | 엔드포인트 |
|------|------|------------|
| 회원가입 | ✅ 완료 | `POST /api/auth/signup` |
| 로그인 | ✅ 완료 | `POST /api/auth/login` |
| 내 정보 조회 | ✅ 완료 | `GET /api/me` |
| 프로필 생성/수정 | ✅ 완료 | `PUT /api/me/profile` |

**인증 방식**: JWT Bearer Token (12시간 유효)  
**비밀번호 암호화**: BCrypt

## 3. API 명세

### 3.1 회원가입

```
POST /api/auth/signup
Content-Type: application/json
인증 불필요
```

**요청 본문**

| 필드 | 타입 | 필수 | 검증 | 설명 |
|------|------|------|------|------|
| loginId | String | ✅ | max=50 | 로그인 ID |
| password | String | ✅ | min=8, max=100 | 비밀번호 |
| phone | String | ✅ | max=30 | 휴대폰 번호 |
| email | String | ✅ | 이메일 형식, max=100 | 이메일 주소 |

```json
{
  "loginId": "hong123",
  "password": "password1234",
  "phone": "010-1234-5678",
  "email": "hong@example.com"
}
```

**응답 (200 OK)**

```json
{
  "userId": 1,
  "loginId": "hong123",
  "accessToken": "eyJhbGciOiJIUzI1NiJ9...",
  "profileCompleted": false
}
```

**에러 코드**

| 상황 | 상태 코드 |
|------|-----------|
| loginId 중복 | 409 Conflict |
| email 중복 | 409 Conflict |
| 필드 검증 실패 | 400 Bad Request |

### 3.2 로그인


**요청 본문**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| loginId | String | ✅ | 로그인 ID |
| password | String | ✅ | 비밀번호 |

```json
{
  "loginId": "hong123",
  "password": "password1234"
}
```

**응답 (200 OK)**

```json
{
  "userId": 1,
  "loginId": "hong123",
  "accessToken": "eyJhbGciOiJIUzI1NiJ9...",
  "profileCompleted": true
}
```

> `profileCompleted`는 사용자의 현재 DB 상태값 반영. 프론트에서 프로필 입력 페이지 라우팅에 활용.

**에러 코드**

| 상황 | 상태 코드 |
|------|-----------|
| loginId 없음 | 401 Unauthorized |
| 비밀번호 불일치 | 401 Unauthorized |
| 소프트 삭제된 계정 | 401 Unauthorized |

## 4. API 흐름도

### 4.1 회원가입 흐름

```
Client
  │
  │ POST /api/auth/signup {loginId, password, phone, email}
  ▼
AuthController.signup(@Valid SignupRequest)
  │
  ▼
AuthService.signup()
  ├─ UserRepository.existsByLoginId() → 중복이면 409
  ├─ UserRepository.existsByEmail()   → 중복이면 409
  ├─ PasswordEncoder.encode(password)
  ├─ UserRepository.save(User)        → status=ACTIVE, role=USER, profileCompleted=false
  └─ JwtTokenProvider.generateToken(userId, loginId, Role.USER)
       └─ Claims: { sub: loginId, userId, loginId, role: "USER", exp: now+12h }
  │
  ▼
AuthResponse { userId, loginId, accessToken, profileCompleted: false }
```

---

### 4.2 로그인 흐름

```
Client
  │
  │ POST /api/auth/login {loginId, password}
  ▼
AuthController.login(@Valid LoginRequest)
  │
  ▼
AuthService.login()
  ├─ UserRepository.findByLoginIdAndDeletedFalse() → 없으면 401
  ├─ PasswordEncoder.matches(입력, hash)           → 불일치면 401
  └─ JwtTokenProvider.generateToken(userId, loginId, user.role)
  │
  ▼
AuthResponse { userId, loginId, accessToken, profileCompleted }
```

---

### 4.3 JWT 인증 흐름 (인증 필요 요청)

```
Client
  │
  │ GET /api/me
  │ Authorization: Bearer eyJhbGci...
  ▼
JwtAuthenticationFilter (OncePerRequestFilter)
  ├─ Authorization 헤더에서 "Bearer " 뒤 토큰 추출
  ├─ JwtTokenProvider.parse(token) → 서명 검증 + 만료 확인
  ├─ Claims에서 userId, loginId, role 추출
  ├─ AuthenticatedUser(userId, loginId) 생성
  ├─ resolveAuthorities(claims)
  │     role="ADMIN" → [ROLE_ADMIN, ROLE_USER]
  │     role="USER"  → [ROLE_USER]
  └─ SecurityContextHolder에 Authentication 설정
  │
  ▼
UserController.me(@AuthenticationPrincipal AuthenticatedUser user)
  │
  ▼
UserProfileService.getMe(userId)
  ├─ UserRepository.findById(userId)
  ├─ UserProfileRepository.findByUserIdAndDeletedFalse(userId)
  └─ UserCategoryRepository.findByUserId(userId)
  │
  ▼
UserProfileMapper.toResponse(user, profile, categories)
  │
  ▼
UserProfileResponse
```

---

### 4.4 프로필 Upsert 흐름

```
Client
  │
  │ PUT /api/me/profile {age, maritalStatus, categories, ...}
  ▼
UserController.upsertProfile(@Valid UserProfileRequest)
  │
  ▼
UserProfileService.upsertProfile(userId, request)
  ├─ UserRepository.findById(userId) → 없으면 404
  ├─ UserProfileRepository.findByUserIdAndDeletedFalse(userId)
  │     없음 → UserProfile 신규 생성 + save()
  │     있음 → profile.update(request 필드 전체)
  ├─ UserCategoryRepository.deleteByUserId(userId)   ← 기존 카테고리 전량 삭제
  ├─ request.categories() 반복 → UserCategory 생성 + save()
  └─ user.markProfileCompleted()                     ← profileCompleted = true
  │
  ▼
getMe(userId) 재호출 → UserProfileResponse 반환
```

## 6. JWT 설계

### Claims 구조

| 클레임 | 값 | 설명 |
|--------|-----|------|
| sub | loginId | JWT 표준 subject |
| userId | Long | 사용자 PK |
| loginId | String | 로그인 ID (중복 저장) |
| role | "USER" / "ADMIN" | 사용자 권한 |
| iat | 발급 시각 (Unix) | issuedAt |
| exp | 발급 + 12시간 (Unix) | expiration |

### 권한 매핑

| DB role | Spring Security 권한 |
|---------|---------------------|
| USER | `ROLE_USER` |
| ADMIN | `ROLE_ADMIN`, `ROLE_USER` |

> ADMIN은 USER 권한도 포함하므로 일반 사용자 API(`/api/me`)도 접근 가능.

### 설정값 (환경변수)

```properties
jwt.secret=${JWT_SECRET}          # .env 파일에서 주입
jwt.expiration-hours=12
```

## 7. SecurityConfig 요약

### 인증 불필요 (permitAll)

| 경로 | 설명 |
|------|------|
| `POST /api/auth/**` | 회원가입, 로그인 |
| `/error` | 스프링 에러 포워딩 (STATELESS 세션에서 인증 컨텍스트 보존용) |
| `GET /api/announcements/**` | 공고 목록/상세 조회 |
| `GET /api/filters/**` | 필터 조회 |
| `OPTIONS /**` | CORS 프리플라이트 |
| `/swagger-ui/**`, `/v3/api-docs/**` | Swagger UI |

### 권한 필요

| 경로 | 필요 권한 |
|------|-----------|
| `/api/admin/**` | `ROLE_ADMIN` |
| 나머지 모든 경로 | 인증 (로그인) 필요 |

### CORS

## 8. Enum 레퍼런스

### CategoryCode (수혜 대상 카테고리)

| 값 | 설명 |
|----|------|
| `YOUTH` | 청년 (만 19~39세, 미혼) |
| `NEWLYWED` | 신혼부부 (혼인 7년 이내) |
| `HOMELESS` | 무주택자 |
| `ELDERLY` | 고령자 (만 65세 이상) |
| `LOW_INCOME` | 저소득자 (기초수급자, 차상위) |
| `MULTI_CHILD` | 다자녀 (미성년 자녀 3인 이상) |

### MaritalStatus (결혼 상태)

| 값 | 설명 |
|----|------|
| `SINGLE` | 미혼 |
| `MARRIED` | 기혼 |
| `OTHER` | 기타 (이혼, 사별 등) |

### Role (사용자 권한)

| 값 | 설명 |
|----|------|
| `USER` | 일반 사용자 (기본값) |
| `ADMIN` | 관리자 |

> 관리자 계정 부여: `UPDATE users SET role = 'ADMIN' WHERE login_id = '계정';`

### UserStatus (계정 상태)

| 값 | 설명 |
|----|------|
| `ACTIVE` | 활성 계정 |
| `INACTIVE` | 비활성 계정 (탈퇴 또는 정지) |
