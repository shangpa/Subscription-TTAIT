# 현행 즐겨찾기 일정 백엔드 DTO / 컨트롤러 정리

작성일: 2026-06-11
최신화: 2026-06-12

## 1. 작업 목적

사용자가 즐겨찾기한 공고를 신청 일정 기준으로 묶어서 조회할 수 있도록 `GET /api/me/favorites/schedule` API를 제공한다.

이 API는 다음 정보를 한 번에 반환한다.

- 상단 요약 카운트
- 상태별 그룹 목록
- 캘린더 이벤트 목록
- 일정 확인 안내 문구

2026-06-12 안정화 작업으로 아래 동작이 추가됐다.

- `Asia/Seoul` 기준 날짜 계산
- 표시 필드 fallback
- DB page 기반 200건 cap
- `summary.returnedCount`, `summary.truncated` 응답 필드
- 일정 우선순위 정렬 후 cap 적용

## 2. 컨트롤러

대상 파일:

`subscrition/src/main/java/com/ttait/subscription/notification/favorite/controller/FavoriteController.java`

### 2-1. 엔드포인트

```http
GET /api/me/favorites/schedule
Authorization: Bearer {accessToken}
```

### 2-2. 역할

- 현재 로그인한 사용자의 즐겨찾기 공고만 조회한다.
- `CurrentUser.id()`로 인증 사용자 ID를 가져온다.
- 관리자 검수 완료 상태의 공개 공고만 일정에 포함한다.
- 일정 상태를 계산해서 `summary`, `groups`, `calendarEvents`, `disclaimer` 형태로 반환한다.

### 2-3. 컨트롤러 메서드

```java
@GetMapping("/schedule")
public ResponseEntity<FavoriteScheduleResponse> schedule() {
    return ResponseEntity.ok(favoriteScheduleService.getSchedule(CurrentUser.id()));
}
```

## 3. 조회와 정렬 정책

대상 파일:

`subscrition/src/main/java/com/ttait/subscription/notification/favorite/repository/UserFavoriteAnnouncementRepository.java`

일정 API는 `findSchedulePageByUserIdWithAnnouncement(...)`를 사용한다.

필터 조건:

- `f.userId = :userId`
- `announcement.deleted = false`
- `announcement.merged = false`
- `AnnouncementEligibility.reviewStatus IN (APPROVED, CORRECTED)`

page 정책:

- service는 `PageRequest.of(0, 200)`으로 조회한다.
- DB query의 `countQuery`로 전체 visible 즐겨찾기 수를 계산한다.
- 실제 응답 item은 최대 200개다.
- 저장 순서가 아니라 일정 우선순위 정렬 후 200건 cap을 적용한다.

정렬 우선순위:

| 순서 | 상태 | 기준 |
|---:|---|---|
| 1 | `DUE_TODAY` | 마감일이 오늘 |
| 2 | `DUE_TOMORROW` | 마감일이 내일 |
| 3 | `DUE_SOON` | 마감일이 7일 이내 |
| 4 | `OPEN` | 접수 중이며 7일 초과 남음 |
| 5 | `UPCOMING` | 접수 시작 전 |
| 6 | `DATE_UNKNOWN` | 시작일/마감일 누락 또는 역전 |
| 7 | `CLOSED` | 마감일이 지남 |

## 4. DTO 정리

DTO 위치:

`subscrition/src/main/java/com/ttait/subscription/notification/favorite/dto`

### 4-1. `FavoriteScheduleResponse`

역할: 일정 조회 API의 최상위 응답 객체

필드:

| 필드 | 설명 |
|---|---|
| `summary` | 상단 요약 정보 |
| `groups` | 상태별 공고 그룹 |
| `calendarEvents` | 일정 캘린더 이벤트 목록 |
| `disclaimer` | 일정 확인 안내 문구 |

### 4-2. `FavoriteScheduleSummaryResponse`

역할: 상단 카운트 요약

필드:

| 필드 | 설명 |
|---|---|
| `totalCount` | cap 적용 전 전체 visible 즐겨찾기 공고 수 |
| `returnedCount` | 현재 응답에 실제 포함된 item 수, 최대 200 |
| `truncated` | `totalCount > returnedCount`이면 `true` |
| `dueTodayCount` | 반환 item 중 오늘 마감 수 |
| `dueTomorrowCount` | 반환 item 중 내일 마감 수 |
| `dueSoonCount` | 반환 item 중 `DUE_TODAY`, `DUE_TOMORROW`, `DUE_SOON` 합계 |
| `openCount` | 반환 item 중 `DUE_TODAY`, `DUE_TOMORROW`, `DUE_SOON`, `OPEN` 합계 |
| `upcomingCount` | 반환 item 중 접수 예정 수 |
| `dateUnknownCount` | 반환 item 중 일정 확인 필요 수 |
| `closedCount` | 반환 item 중 마감됨 수 |

주의:

- `totalCount`만 전체 visible 즐겨찾기 기준이다.
- 나머지 상태별 count는 실제 반환된 최대 200개 item 기준이다.
- `truncated=true`이면 상태별 count가 전체 즐겨찾기 상태 분포를 의미하지 않는다.

### 4-3. `FavoriteScheduleGroupResponse`

역할: 상태별 묶음 단위

필드:

| 필드 | 설명 |
|---|---|
| `key` | 상태 키 |
| `label` | 화면 표시용 라벨 |
| `items` | 해당 상태에 속한 공고 목록 |

그룹 순서:

1. `DUE_TODAY`
2. `DUE_TOMORROW`
3. `DUE_SOON`
4. `OPEN`
5. `UPCOMING`
6. `DATE_UNKNOWN`
7. `CLOSED`

### 4-4. `FavoriteScheduleItemResponse`

역할: 그룹 내부의 개별 공고 일정 정보

필드:

| 필드 | 설명 |
|---|---|
| `announcementId` | 공고 ID |
| `noticeName` | 공고명 |
| `providerName` | 공급 기관명 |
| `noticeStatus` | 공고 상태 |
| `applicationStartDate` | 신청 시작일 |
| `applicationEndDate` | 신청 마감일 |
| `announcementDate` | 공고일 |
| `winnerAnnouncementDate` | 당첨자 발표일 |
| `favoritedAt` | 즐겨찾기 등록 시각 |
| `dDay` | 마감일까지 남은 일수 |
| `dDayLabel` | `D-day`, `D-7`, `마감` 같은 문자열 |
| `scheduleStatus` | 일정 상태 enum |
| `statusMessage` | 상태 설명 문구 |
| `actionLabel` | 현재 고정값 `공고 확인` |

표시 필드 fallback:

| 필드 | fallback |
|---|---|
| `noticeName` | `공고명 미확인` |
| `providerName` | `기관 미확인` |
| `noticeStatus` | `UNKNOWN` |

### 4-5. `FavoriteCalendarEventResponse`

역할: 캘린더에 바로 표시할 수 있는 일정 이벤트

필드:

| 필드 | 설명 |
|---|---|
| `announcementId` | 공고 ID |
| `eventType` | 이벤트 종류 |
| `date` | 일정 날짜 |
| `title` | 화면 표시용 제목 |
| `priority` | 우선순위 |

현재 사용하는 `eventType`:

- `ANNOUNCEMENT_DATE`
- `APPLICATION_START`
- `APPLICATION_END`
- `WINNER_ANNOUNCEMENT`

현재 사용하는 `priority`:

- `HIGH`
- `NORMAL`
- `LOW`

### 4-6. `FavoriteScheduleStatus`

역할: 공고의 일정 상태를 분류하는 enum

값:

| 값 | 설명 |
|---|---|
| `DUE_TODAY` | 오늘 마감 |
| `DUE_TOMORROW` | 내일 마감 |
| `DUE_SOON` | 7일 이내 마감 |
| `OPEN` | 접수 중 |
| `UPCOMING` | 접수 예정 |
| `DATE_UNKNOWN` | 일정 확인 필요 |
| `CLOSED` | 마감됨 |

## 5. 상태 계산 기준

상태 계산은 `FavoriteScheduleService`에서 수행한다.

날짜 기준:

- `Asia/Seoul` 기준 `LocalDate.now(clock)`을 사용한다.
- 테스트에서는 고정 `Clock`으로 기준일을 주입할 수 있다.

상태 기준:

| 조건 | 상태 |
|---|---|
| 시작일 또는 마감일이 없음 | `DATE_UNKNOWN` |
| 시작일이 마감일보다 뒤 | `DATE_UNKNOWN` |
| 오늘이 마감일보다 뒤 | `CLOSED` |
| 오늘이 시작일보다 앞 | `UPCOMING` |
| 마감일까지 0일 | `DUE_TODAY` |
| 마감일까지 1일 | `DUE_TOMORROW` |
| 마감일까지 7일 이하 | `DUE_SOON` |
| 그 외 접수 기간 내 공고 | `OPEN` |
