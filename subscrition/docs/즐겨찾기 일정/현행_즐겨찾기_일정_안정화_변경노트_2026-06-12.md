# 현행 즐겨찾기 일정 안정화 변경 노트

작성일: 2026-06-12

## 1. 변경 목적

`GET /api/me/favorites/schedule` API가 실제 운영 데이터에서도 안정적으로 동작하도록 아래 리스크를 보완했다.

| 항목 | 기존 리스크 | 변경 결과 |
|---|---|---|
| 날짜 기준 | 서버 기본 timezone에 따라 `DUE_TODAY`, `DUE_TOMORROW`, `CLOSED` 판정이 달라질 수 있음 | `Asia/Seoul` 기준 `Clock` 사용 |
| 표시 필드 | `noticeStatus` 등 일부 값이 비정상일 때 500 가능 | 응답 매핑 시 fallback 적용 |
| 대량 즐겨찾기 | 응답/DB 조회가 커질 수 있고 cap 전 정렬이 보장되지 않음 | DB page + 일정 우선순위 정렬 + 200건 cap 적용 |
| 테스트 | service 단위 검증 중심 | service, controller, repository, HTTP E2E 검증 추가 |

## 2. 주요 코드 변경

### 2-1. `FavoriteScheduleService`

대상 파일:

- `subscrition/src/main/java/com/ttait/subscription/notification/favorite/service/FavoriteScheduleService.java`

변경 내용:

- `ZoneId.of("Asia/Seoul")` 기준 `Clock`을 사용한다.
- 운영 경로는 `LocalDate.now(clock)`으로 오늘 날짜를 계산한다.
- 테스트에서는 package-private 생성자로 고정 `Clock`을 주입할 수 있다.
- 일정 응답은 최대 `200`개만 반환한다.
- `totalCount`는 cap 전 전체 visible 즐겨찾기 수다.
- `returnedCount`는 실제 응답에 포함된 item 수다.
- `truncated`는 `totalCount > returnedCount`일 때 `true`다.
- `noticeName`, `providerName`, `noticeStatus`는 응답 매핑 시점에 fallback 처리한다.

fallback 기준:

| 필드 | 조건 | 응답값 |
|---|---|---|
| `noticeName` | `null` 또는 blank | `공고명 미확인` |
| `providerName` | `null` 또는 blank | `기관 미확인` |
| `noticeStatus` | `null` | `UNKNOWN` |

### 2-2. `UserFavoriteAnnouncementRepository`

대상 파일:

- `subscrition/src/main/java/com/ttait/subscription/notification/favorite/repository/UserFavoriteAnnouncementRepository.java`

추가 메서드:

- `findSchedulePageByUserIdWithAnnouncement(...)`

역할:

- `userId` 기준으로 현재 사용자 즐겨찾기만 조회한다.
- 삭제/병합 공고를 제외한다.
- `APPROVED`, `CORRECTED` 검수 상태 공고만 포함한다.
- `Pageable`로 DB 수준에서 200건 cap을 적용한다.
- `countQuery`로 전체 visible 즐겨찾기 수를 계산한다.
- 일정 우선순위 정렬 후 page를 반환한다.

정렬 우선순위:

1. `DUE_TODAY`
2. `DUE_TOMORROW`
3. `DUE_SOON`
4. `OPEN`
5. `UPCOMING`
6. `DATE_UNKNOWN`
7. `CLOSED`

보조 정렬:

- 접수 예정은 `applicationStartDate` 오름차순
- 그 외 일정은 `applicationEndDate` 오름차순
- 일정 미확인 공고는 즐겨찾기 등록일 내림차순
- 최종 tie-breaker는 공고명, 공고 ID

### 2-3. `FavoriteScheduleSummaryResponse`

대상 파일:

- `subscrition/src/main/java/com/ttait/subscription/notification/favorite/dto/FavoriteScheduleSummaryResponse.java`

추가 필드:

| 필드 | 의미 |
|---|---|
| `returnedCount` | 현재 응답에 실제 포함된 item 수 |
| `truncated` | 전체 visible 즐겨찾기 중 일부만 반환됐는지 여부 |
