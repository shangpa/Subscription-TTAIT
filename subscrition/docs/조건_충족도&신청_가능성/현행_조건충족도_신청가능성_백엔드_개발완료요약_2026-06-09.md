이 문서는 공고 상세 화면에서 사용할 “내 조건 충족도 / 신청 가능성 체크리스트” backend 개발 완료 범위를 정리한다.

| 항목 | 상태 | 내용 |

## 4. API 계약

```http
GET /api/announcements/{announcementId}/eligibility-checklist
Authorization: Bearer {USER_TOKEN}

HTTP/1.1 200 OK
Cache-Control: private, no-store
```

주의:

- `SecurityConfig`상 `GET /api/announcements/**`는 public permit일 수 있지만, 이 endpoint는 controller 내부에서 `CurrentUser.id()`를 호출하므로 JWT가 없으면 `401 authentication required`가 반환된다.
- 공고가 public-visible 상태가 아니면 `404 announcement not found`가 반환된다.
- 이 응답은 사용자 프로필 기반 개인화 결과이므로 `Cache-Control: private, no-store`를 내려 공용 cache와 브라우저 저장을 방지한다.

## 6. 상태값

### 6.1 항목 상태

| 값 | 의미 |
| - | - |
| `MET` | 저장된 프로필 기준 조건 충족 |
| `NOT_MET` | 저장된 프로필 기준 조건 미충족 |
| `NEEDS_VERIFICATION` | 원문 또는 프로필 보완 확인 필요 |
| `NOT_APPLICABLE` | 공고에 해당 조건이 없거나 MVP 자동 판정 대상이 아님 |

### 6.2 요약 상태

| 값 | 의미 |
| - | - |
| `LIKELY_READY` | 명확한 미충족/확인 필요 없이 주요 조건 충족 |
| `REVIEW_REQUIRED` | 미충족은 없지만 확인 필요 항목 존재 |
| `HAS_BLOCKERS` | 하나 이상의 `NOT_MET` 존재 |
| `INSUFFICIENT_DATA` | eligibility 데이터 부족 |

## 7. 판정 항목

| key | group | label | 판정 기준 |
| - | - | - | - |
| `AGE` | 기본 자격 | 나이 | `UserProfile.age`와 `AnnouncementEligibility.ageMin/ageMax` 비교 |
| `MARITAL` | 가구 조건 | 혼인 상태 | `maritalStatus`와 `maritalTargetType` 비교 |
| `NEWLYWED` | 가구 조건 | 신혼부부 | `marriageYears`와 `marriageYearLimit` 비교 |
| `CHILDREN` | 가구 조건 | 자녀 수 | `childrenCount`와 `childrenMinCount` 비교 |
| `HOMELESS` | 주택 보유 | 무주택 | `homelessRequired=true`이면 `profile.homeless` 확인 |
| `LOW_INCOME` | 소득·자산 | 저소득/수급 | `lowIncome`, `recipient`, `nearPoverty` 중 하나라도 true면 충족 |
| `ELDERLY` | 기본 자격 | 고령자 | `elderly=true` 또는 기준 나이 이상이면 충족 |
| `DEPOSIT_BUDGET` | 비용 | 보증금 예산 | 공고 보증금이 프로필 최대 보증금 이하인지 확인 |
| `MONTHLY_RENT_BUDGET` | 비용 | 월세 예산 | 공고 월세가 프로필 최대 월세 이하인지 확인 |
| `APPLICATION_PERIOD` | 일정 | 신청 기간 | 현재 날짜가 접수 기간 안이면 충족, 접수 전은 확인 필요 |
| `INCOME_ASSETS` | 소득·자산 | 소득·자산 상세 기준 | raw 기준 존재 시 원문 확인 필요, raw text는 미노출 |
